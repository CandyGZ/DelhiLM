# Contributing to Delphi Pipeline

First off, thank you for considering contributing to the Delphi Pipeline project! Your help is greatly appreciated. Every contribution helps to make this project better.

This document provides some guidelines for contributing.

## How Can I Contribute?

### Reporting Bugs

If you find a bug, please ensure it hasn't already been reported by searching on GitHub under [Issues](https://github.com/your-username/DelhiLM/issues).

If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/your-username/DelhiLM/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample** or an **executable test case** demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements

If you have an idea for an enhancement, please open a new issue to discuss it. This allows us to coordinate efforts and ensure the enhancement aligns with the project's goals.

- Provide a clear and detailed explanation of the feature.
- Explain why this enhancement would be useful.
- If you are willing to implement it, please mention it in the issue.

### Pull Requests

We welcome pull requests for bug fixes, new features, and improvements to the documentation.

1.  **Fork the repository**: Click the 'Fork' button at the top right of this page.
2.  **Clone your fork**: `git clone https://github.com/your-username/DelhiLM.git`
3.  **Create a new branch**: `git checkout -b feature/your-feature-name` or `git checkout -b fix/your-bug-fix-name`. Please use a descriptive branch name.
4.  **Make your changes**: Make your changes in the new branch. Ensure your code follows the existing style.
5.  **Commit your changes**: Commit your changes with a clear and descriptive commit message. We recommend following the Conventional Commits specification. For example:
    ```bash
    git commit -m "feat: Add new visualization for Brier scores"
    ```
6.  **Push to your fork**: `git push origin feature/your-feature-name`
7.  **Create a Pull Request**: Go to the original repository on GitHub and create a new pull request from your forked branch. Provide a clear title and description for your pull request, explaining the changes you've made.

## Styleguides

### Code Style

-   Please try to follow the existing code style in the `Delphi_LLMs.ipynb` notebook to maintain consistency.
-   Use clear and descriptive variable and function names.
-   Add comments to your code where necessary to explain complex logic.

### Commit Messages

-   As mentioned, we recommend using the Conventional Commits format.
-   This helps in generating automated changelogs and makes the commit history more readable. Common types include `feat:`, `fix:`, `docs:`, `style:`, `refactor:`, `test:`.
-   If your commit resolves an open issue, reference it in your commit message using keywords like `closes`, `fixes`, or `resolves` followed by the issue number (e.g., `resolves #45`). This will automatically close the issue when the commit is merged.

## Code of Conduct

This project and everyone participating in it is governed by the Code of Conduct. By participating, you are expected to uphold this code. Please report unacceptable behavior.
